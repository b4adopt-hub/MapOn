# MapOn 아키텍처

AI 기반 토지 검증·활용 시뮬레이션·업체 매칭 플랫폼.

## 전략: 단일 도구 훅 → 양면 플랫폼

콜드스타트(닭-달걀) 해소를 위해 소비자용 단일 도구로 트래픽을 먼저 만들고,
축적된 수요 데이터를 미끼로 B2B 업체를 끌어들인다.

### 1차 훅 (현재 단계)
- 토지 정보분석 카드: 지번 입력 → 용도지역·지목·공시지가·규제·목적별 가능성 등급
- 2D 경계 오버레이: 지도 위 GPS 현위치 + 지적 경계 폴리곤 (참고용, 법적 경계 아님)
- 비용 0 원칙: 무상 공공 API + 룰 엔진만. AI 실사화·SSR·LLM 미사용.
- 소셜 간편가입: Google / Kakao / Email(직접입력)

### 2차 이후
- 업체(B2B) 매칭·크레딧·제안서·계약폼·보증 패킷
- 경계 정밀화: SSR(국토지리정보원) 연동, 개방 평지 1m 이내
- AI 실사화(크레딧 유료)

## 데이터 3축 (모두 국가 무상 제공)

| 축 | 출처 | 제공 |
|---|---|---|
| 공간정보 | 브이월드(VWorld) | 지적도·용도지역·지목·경사(DEM)·배수·공시지가 |
| 규제·인허가 | 토지이음(LURIS) | 토지이용계획·지역지구·개발행위허가 |
| 고정밀 위치 | 국토지리정보원 SSR | GNSS 보정(2차 단계) |

## 데이터 레이어 (Supabase, ap-northeast-2)

- `profiles`: auth.users 확장. 인증경로 Google/Kakao/Email.
- `land_lookups`: PNU 기준 토지 조회 캐시. 공용 읽기, service_role 쓰기.
- `diagnoses`: 사용자 진단 결과. 본인만 접근(RLS).
- `demand_signals`: 익명 수요 집계. 개인정보·지번 미포함. B2B 영업 자산.

## 보안 원칙 (b4adopt/ALCU 계승)

- 전 테이블 RLS 활성화. 보안 자문 경고 0건 유지.
- 함수 search_path 고정, 트리거 함수 REST 노출 차단.
- service_role/공공 API 키는 Edge Function 시크릿로만 보관. 프론트 노출 금지.
- 경계 출력은 항상 "참고용·법적 경계 아님" 표기 (측량 오해 리스크 차단).
